# Ice Tea Usage Guide

Ice Tea is designed to be highly configurable via the CLI, environment variables, and its configuration file (`.ice-tea.yaml`).

## Basic Invocation

To scan a directory:
```bash
./ice-tea scan ./my-project
```

To scan a specific file:
```bash
./ice-tea scan ./main.go
```

## Global Flags

These flags can be used with any subcommand:
- `--config <path>`: Path to a custom config file (default: `.ice-tea.yaml`)
- `--verbose, -v`: Enable verbose debug logging
- `--log-level <level>`: Set exact log level (info, warn, error, debug, fatal)
- `--no-color`: Disable ANSI color output

## `scan` Subcommand Flags

The `scan` command is the core of Ice Tea. Use these flags to control the scanning process:

| Flag | Short | Description | Default |
|------|-------|-------------|---------|
| `--format` | `-f` | Output format (`console`, `json`, `sarif`, `gitlab`) | `console` |
| `--output` | `-o` | Output file path (if empty, writes to stdout) | `""` |
| `--severity` | `-s` | Minimum severity threshold (`info`, `low`, `medium`, `high`, `critical`) | `medium` |
| `--confidence`| | Minimum confidence threshold (`low`, `medium`, `high`) | `medium` |
| `--exclude-dir`| | Comma separated list of directories to ignore | `node_modules,vendor,.git` |
| `--exclude-file`| | Comma separated wildcard matchers (e.g., `*test.go`) | `*.test` |
| `--language` | | Restrict scanning exclusively to these languages (comma separated) | `[]` (Auto) |
| `--concurrency`| `-c` | Number of worker routines to use for processing | `4` |
| `--skills-dir` | | Custom directory to load SKILL definitions from | `""` |
| `--enable-llm` | | Enable LLM deep reasoning for false-positive filtering | `false` |

## Integrating with LLM Deep Reasoning

Ice Tea's Engine 3 utilizes Large Language Models to evaluate findings generated by static patterns. This drastically reduces false positives.

1. Ensure your chosen provider requires less configuration (currently OpenAI is prioritized).
2. Export your API key:
   ```bash
   # Unix/Linux/macOS
   export OPENAI_API_KEY="sk-proj-xyz..."

   # Windows PowerShell
   $env:OPENAI_API_KEY="sk-proj-xyz..."
   ```
3. Run the scan with the LLM flag:
   ```bash
   ./ice-tea scan . --enable-llm --format console
   ```

## Configuration File (`.ice-tea.yaml`)
For CI/CD and large projects, having a configuration file is much easier than passing 20 arguments to the CLI.

Place `.ice-tea.yaml` in the root of your project:
```yaml
# Output Settings
output:
  format: "sarif"
  file: "results.sarif"
  color: true

# Scanner Settings
scan:
  severity: "high"
  confidence: "medium"
  concurrency: 8

# False-Positive Engine Context
llm:
  enabled: true
  provider: "openai"
  model: "gpt-4o"

# Exclusions
exclude:
  dirs: ["vendor", "node_modules", ".git", "testdata", "dist"]
  files: ["*.min.js", "*_test.go", "*.generated.go"]
```

## Exit Codes

Ice Tea respects standard unix exit codes, making it perfect for CI/CD gates.

- `0`: Completed successfully. No vulnerabilities found above the specified `severity` threshold.
- `1`: Scan completed, but vulnerabilities **were found** above the severity threshold.
- `2`: Fatal error (e.g. invalid arguments, target doesn't exist, configuration error).
